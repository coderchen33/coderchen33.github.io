1. CSS简介

1.1 什么是CSS

CSS 指层叠样式表 (Cascading Style Sheets) 用来定义如何显示HTML元素。CSS不仅可以静态地修饰网页，还可以配合各种脚本语言动态地对网页各元素进行格式化。

我们之前学习了html可以用来在网页上内容，但是这些内容是黑白的，或者即便有样式也是非常简陋的，这样的网页放到现在没有一个人会看。当然我们可以在html中添加一些样式，但是随着网页的丰富，样式越来越多会造成html越来越复杂。并且假如我们有1000个h1标签 打算修改所有的样式，如果只在html中修改，需要每个修改一次，总共1000次。当然这么多重复性的工作，没人愿意做的。 另外，现在的网页布局都是十分合理的，像导航栏，侧边栏，网页底部等都是通过css来进行定位的。

为什么要有CSS?

CSS就是为了解决上面提到的问题，它的优点主要有：

  • 解决内容与表现分离（html和css分离）的问题
  • 通过选择器批量或者精确的修改标签样式，提高工作效率
  • 通过浮动和定位对网页进行精确布局

2.4 属性选择器

属性选择器可以根据元素的属性及属性值来选择元素

  • 简单属性选择 [属性]{}
  • 根据具体属性值选择 [属性=属性值]{} 属性与属性值必须完全匹配
  • 根据部分属性值选择 [属性~=属性值]{}
  • 部分属性时也可以使用子串匹配属性
    • [abc^="def"] 选择 abc 属性值以 "def" 开头的所有元素
    • [abc$="def"] 选择 abc 属性值以 "def" 结尾的所有元素
    • [abc*="def"] 选择 abc 属性值中包含子串 "def" 的所有元素

4. CSS盒子模型

CSS 框模型 (Box Model) 规定了元素框处理元素内容、内边距、边框和 [外边距]的方式。


所有和盒子模型都包含上面的几部分：

  • margin ： 外边距 边框外面就是外边距
  • border： 边框 内边框的边缘结束就是边框 边框可以有宽度
  • padding：内边距 直接包围元素内容
  • element: 真正的元素内容 （width 宽度 height 高度）

注意：

  • 在 CSS 中，width 和 height 指的是内容区域的宽度和高度。增加内边距、边框和外边距不会影响内容区域的尺寸，但是会增加元素框的总尺寸。
  • 背景应用于由内容和内边距、边框组成的区域。
  • 外边距可以是负值，而且在很多情况下都要使用负值的外边距。

5. CSS浮动

5.1 标准文档流

文档流指的是元素排版布局过程中，元素会默认自动从左往右，从上往下的流式排列方式。并最终窗体自上而下分成一行行，并在每行中从左至右的顺序排放元素。

还记不记得我们到现在为止写的html页面，如果是块级元素的话，是不是每个块级元素自己占一行，然后下面的块级元素只能另起一行，这就是从上往下。 如果是内联元素，也就是行内元素的话，是不是一直占满整个页面后再换一行，这就是从左到右。

但是现在的网页的布局都是按块来进行的，如何让块和块之间能够并列在一行，或者如何让行内元素排列成一列，这就是我们需要考虑的问题。


标准文档流等级

*块级元素 *

  • 独占一行，不能与其他任何元素并列
  • 能接受宽、高
  • 如果不设置宽度，那么宽度将默认变为父亲的100%，即和父亲一样宽

内联元素（行内元素）

  • 与其他元素并排
  • 不能设置宽、高。默认的宽度就是文字的宽度

内联元素可以被包含在块级元素中，反之则不可以。

5.2 display

如果需要实现块内元素和行内元素的相互转换，则需要使用display属性。

display的属性值：

  • inline ： 行内元素
  • block： 块级元素
  • inline-block ： 块级元素，但是可以级联
  • none ： 不显示 ， 但是仍然存在

5.3 float

如果想要让行内元素和块级元素不在遵循于本身的特性，除了转换(display)以外，还可以使用float 以及 position

float（浮动） 和 position(定位 ) 可以让元素脱离”标准文档流”。

float属性：

  • 左浮右浮 float：left / right

6. CSS定位

6.1 相对定位

相对定位就是相对于自己原来的位置来进行指定的偏移 相对定位的话仍然在标准文档流中，且原位置被保留。


6.2 绝对定位

绝对定位是基于xxx进行定位 方向也是上下左右四个方向 绝对定位后元素不在标准文档流中，且原位置不保留

基于xxx定位的意思是说：

  • 如果父级元素没有定位，则相对于浏览器进行定位。
  • 如果父级元素进行了定位，则相对于父级元素进行定位

6.4 z-index

z-index 属性设置元素的堆叠顺序。拥有更高堆叠顺序的元素总是会处于堆叠顺序较低的元素的前面，

默认是从0开始排序。
