1. Mybatis简介

1.1什么是Mybatis


首先我们先看下它的官方文档是如何介绍Mybatis的：

  • MyBatis 是一款优秀的持久层框架，它支持自定义 SQL、存储过程以及高级映射。
  • MyBatis 免除了几乎所有的 JDBC 代码以及设置参数和获取结果集的工作。
  • MyBatis 可以通过简单的 XML 或注解来配置和映射原始类型、接口和 Java POJO为数据库中的记录。

1.2 为什么使用Mybatis

  • 是Mybatis是ORM的一种实现框架，都是对JDBC的一种封装！（回想一下我们之前写代码直接操作原生jdbc的时候每次都要写一堆代码才能拿到connection对象，所以我们封装了jdbcUtils工具类。而Mybatis则是对JDBC的一种更高级的封装）

  • sql和代码分离，提高了代码的可维护性。

  • 支持动态sql,避免了之前动态sql的拼接。

  • 社区活跃，使用人数多


1.3 Mybatis相关资料

  • 官方文档：https://mybatis.org/mybatis-3/zh/index.html
  • github:https://github.com/mybatis/mybatis-3
  • maven仓库：https://mvnrepository.com/

我们之后的学习也是以官方文档为准，进行学习。

这里可能会有一个问题，导致最后结果执行不成果。就是maven的资源过滤问题。

  • 如果上面写的映射文件是直接和接口一样放到了mapper包下,就会报错。(maven是约定大于配置，xml等配置文件应该放在resources文件夹下。放到mapper包下则不会被导出) (不推荐)
  • 如果映射文件是在resources下新建的mapper包下，则不会出问题。 (推荐，因为xml文件是配置文件，所以放到resources下比较合理，但是需要注意也要建包com.company.mapper，放到mapper包下)。

对于映射文件直接放在mapper包下的，需要在pom文件中配置maven导出资源的位置。

4. 注解实现CRUD(了解即可)

对于一些简单的sql语句，我们甚至可以省略映射文件，直接使用注解进行开发。但是，需要注意

  • 注解开发只适用于一些简单的sql，对于复杂的sql显的力不从心。
  • 虽然省略了映射文件，但是还是需要在mappers中进行注册。
  • 之后绝大部分时候还是使用配置文件的方式，所以注解了解即可。

注解直接加在接口方法上，可以省略映射文件。

但是不要忘了注册（没有xml文件，所以可以使用类路径的方式进行注册）
