1.JSP简介

1.1 什么是JSP

JSP（JavaServer Pages) Java服务器端页面:

  • 和Servlet一样也是一种动态网页开发技术，使用JSP标签在HTML网页中插入Java代码。
  • JSP是一种特殊的Java servlet，主要用于实现Java web应用程序的用户界面部分。
  • 网页开发者们通过结合HTML代码、XHTML代码、XML元素以及嵌入JSP操作和命令来编写JSP。

为什么要有JSP？

回想一下，我们在学完servlet后，想要给前端显示html页面是如何做的?

是不是直接在Servlet中直接写了整个html页面，这样做的问题非常多：一方面，每次都要写out.println非常繁琐，另一方面，如果html页面非常庞大，这样写非常容易出问题，比如少写了<等等。

为了解决上面这些问题才有了JSP页面，它用来将servlet和用户界面相互分离。servlet负责处理请求，JSP负责用户页面展示。比如上面的这个直接使用JSP页面展示:

3.JSP内置对象和作用域

九大内置对象

pageContext PageContext类的实例，提供对JSP页面所有对象以及命名空间的访问
request HttpServletRequest 接口的实例
response HttpServletResponse 接口的实例
session HttpSession类的实例
application ServletContext类的实例，与应用上下文有关
config ServletConfig类的实例
out JspWriter类的实例，用于把结果输出至网页上
page 类似于Java类中的this关键字
Exception Exception类的对象，代表发生错误的JSP页面中对应的异常对象

这几个对象好多我们已经用过了，这里就介绍下四个存东西对象的作用域，我们之后也是主要使用这几个对象。

  • PageContext 保存的数据只在一个页面中有效
  • request 保存的数据在一次请求中有效（可以请求转发）
  • Session 保存的数据在一次会话中有效 打开浏览器到关闭浏览器
  • Application 保存的数据在服务器中有效 打开服务器到关闭服务器

作用域

PageContext-->request-->session-->application 有点像jjvm中的双亲委派机制。 从pagecontext中找数据时，如果有就找到没有就向上一层一层的找。

4.1 EL表达式

EL（Expression Language） 是为了使JSP写起来更加简单。 语法很简单${} 。作用：

  • 获取数据
  • 执行运算
  • 获取web开发的常用对象

4.3 JSTL表达式

JSTL（Java server pages standarded tag library，即JSP标准标签库） 封装了JSP的通用核心功能。可以利用这些标签取代JSP页面上的Java代码，从而提高程序的可读性，降低程序的维护难度。

根据JSTL标签所提供的功能，可以将其分为5个类别。

  • 核心标签 （只需要掌握部分即可，其他标签都不用了解）
  • 格式化标签
  • SQL 标签
  • XML 标签
  • JSTL 函数
